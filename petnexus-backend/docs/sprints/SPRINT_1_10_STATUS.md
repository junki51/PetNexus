## Overall status

Implemented for demo/learning:

- Backend health and DB health.
- PostgreSQL config and startup migration.
- Auth, bcrypt, JWT.
- Owner profile.
- Breed catalog and owner pets.
- Clinic profile.
- Public pet ID and clinic lookup.
- Appointment foundation.
- Clinic patient derivation from appointments.
- Clinic-owned medical records.

Not implemented:

- Verified clinic onboarding.
- Owner approval/access grants.
- QR authorization.
- Staff management.
- Visits/timeline.
- Notifications.
- Payments.
- Reports/dashboards.
- File uploads/labs/vaccination/prescription sub-records.
- Audit logs.
- Frontend.

Highest current risks:

1. Default JWT secret can be used if production env is misconfigured.
2. Public clinic role registration.
3. Clinic-created appointment can establish patient relationship without owner approval.
4. Medical record response exposes full owner phone.
5. No real PostgreSQL integration test suite.
